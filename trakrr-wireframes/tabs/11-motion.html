Motion specs

Motion has a job. We use it to celebrate (PB), signal urgency (REDLINE), orient (sheet enter/exit), and reward effort (counter rolls into a new time). Never decoration. Hover each tile to replay.

Curves · the only 4 you'll need11.1

Standard · ui

Sheet enter, tab switch, secondary state change. Apple's standard ease — quick start, soft land.
cubic-bezier(.32, .72, 0, 1)240ms

Spring · celebrate

PB badge, finish stamp, follow confirmation. Overshoot ~8% — feels like a small victory.
cubic-bezier(.34, 1.56, .64, 1)420–600ms

Slam · urgent

REDLINE alert, station-done confirmation. Starts at 2x scale, snaps in fast — your eye can't miss it.
cubic-bezier(.6, 0, .8, .4)180–280ms

Linear · numbers

Counter rolls, stopwatch ticks, scrubber. Anything time-coded uses pure linear — no easing on truth.
linearmatches data rate

What we don't animate11.3

Race numbers, mid-race

The pace, HR, and clock during a race are plain text. No tween, no ease. Reading at 180bpm needs the number to be still — motion is noise.
no transition

Settings toggles

iOS handles these. Don't reinvent.
system default

Empty states

Quiet. The void is intentional. Don't fill it with friendly motion.
static

Errors

Snap in, hold, dismissed by user. No fade. We want them seen.
no exit
